## 2024-05-24 - [Avoid Vite Environment Variable Static Replacement Issues]
**Vulnerability:** Replacing `innerHTML` with `textContent` avoids invoking the browser's HTML parser and prevents static security scanners or strict linters from flagging the file for potentially unsafe DOM manipulation.
**Learning:** Vite relies on static string replacement for environment variables, looking exactly for the syntax `import.meta.env.VITE_SOME_KEY`. Accessing `import.meta.env` dynamically (e.g., `(import.meta as any).env`) breaks production builds. Additionally, assigning environment variables to the global `window` object in `src/demo/main.tsx` is an intentional project hack for testing and bundler-agnostic passing; do not remove or refactor this pattern.
**Prevention:** Always verify how Vite builds and replaces environment variables when working with environment variables, do not use optional chaining with it.
